Developer notes

CSV import and live data need the backend running. Use this page when something fails (HTML instead of JSON, port errors, etc.).

1. Start the server

  1. Open Terminal (or Cursor’s terminal).
  2. Go to the backend folder inside this project, then run: 
    npm start
  3. You should see: Paid media backend listening on http://localhost:5001
  4. Leave that terminal open while you use the dashboard.

Open this dashboard at: http://localhost:5001 — same port as the API (avoids “wrong server” errors).

2. Port 5001 already in use

Something else is using the port (often an old Node process).

lsof -i :5001

Note the PID (second column), then run kill and that number, e.g. kill 17616 (use your number).

If it won’t stop: kill -9 and the same number. Then npm start again.

Or use another port: PORT=5002 npm start and open http://localhost:5002.

3. Import shows HTML instead of JSON

  • Confirm the backend is running (npm start).
  • Use http://localhost:5001 in the browser (not a file:// page if imports fail).
  • After changing server code, restart the backend (Ctrl+C, then npm start).

4. Import validation errors (400)

  • Use Download template for correct headers.
  • Include a header row and at least one data row.

5. Stop the server

In the terminal where npm start is running: Ctrl+C.
